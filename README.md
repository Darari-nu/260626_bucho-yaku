# 部長訳

カタカナの未来を、部長に通る言葉へ。

「部長訳」は、IT/AIのカタカナ語をJTC（日系大企業）の上司世代に伝わる和語へ変換し、社内で企画を前に進めるための説明文まで整える静的Webツールです。

お悩み解決サイト選手権の応募作です。締切は2026-06-30、公開URLはGitHub Pages設定後に追記します。

## 概要

技術記事や企画メモを貼ると、辞書ベースでカタカナ語を検出し、次の3層で出力します。

1. 部長向け1行サマリー
2. 稟議に書ける効果文
3. 想定ツッコミ3つと先回り回答

単なる翻訳ツールではなく、未来を社内に持ち込むための変換装置として作っています。

## 本質悩み

表面的な悩みは「AI記事やDX資料を共有しても、カタカナ用語が多すぎて上司に伝わらない」ことです。

本質は、自分は技術の価値を分かっているのに、上司に伝わらないせいで前に進まない孤独です。部長訳は、反対される前提で「何が楽になるのか」「費用対効果はあるのか」「誰が責任を持つのか」まで先に言葉へ変えます。

## 使い方

1. `index.html`をブラウザで開く、またはローカルサーバーでこのディレクトリを配信します。
2. 入力欄に技術記事、社内共有メモ、企画メモを貼ります。
3. `部長訳する`を押します。
4. 3層出力をコピーして、Teams、Slack、メール、稟議下書きに貼ります。
5. 足りない言葉は「この場で辞書を足す」からセッション内だけ追加できます。

ローカルで`fetch`が制限されるブラウザでは、次のように簡易サーバー経由で確認してください。

```bash
python3 -m http.server 8000
```

## 技術構成

- Vanilla HTML/CSS/JavaScript
- 外部APIなし
- CDNなし
- ビルドツールなし
- 辞書データ: `data/dictionary.json`
- 初期辞書: 50語
- 入力処理: ブラウザ内完結

## データ保護方針

- 入力文は外部送信しません。
- サーバー保存、ログ保存、Cookie保存はしません。
- 生成AI API、解析タグ、広告タグは使いません。
- ユーザー追加辞書はv1では現在のブラウザセッション中だけ反映します。
- 画面描画ではユーザー入力を`innerHTML`へ直接入れず、`textContent`とDOM生成で扱います。

## darariの一次情報注記

辞書の言い換えは、darariが製造業JTCのAI推進担当として実際に社内説明で通してきた感覚をもとにしています。

採用基準は、用語としての厳密な正確さよりも「部長が判断できる業務語になっているか」です。業務利用時は、各社の文脈、規程、承認プロセスに合わせて調整してください。

## 辞書改善ループ

公開後は、感想や利用メモから追加語候補を集め、次の観点で辞書を100語化します。

- 面白い横文字ではなく、会議や稟議で止まりやすい言葉か
- 部長訳が具体的な業務変化になっているか
- 効果文が「効率化できます」だけで終わっていないか
- 想定ツッコミに費用対効果、現場利用、責任所在への回答が含まれるか
- 追加語をサンプル文に入れたとき、3層出力が自然に効くか

追加時の確認手順:

1. `data/dictionary.json`に語、訳、例文、カテゴリ、効果文、想定ツッコミを追加します。
2. サンプル文に追加語を入れて変換します。
3. 置換本文、部長向け1行、稟議効果、想定ツッコミを確認します。
4. 仕事中に開ける表現か、軽すぎないかを見直します。
