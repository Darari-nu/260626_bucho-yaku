# デザイン設計 — 部長訳

## デザイン4原則の方針

### 近接
入力、変換、出力の関係が一目で分かる配置にする。

- 入力欄と「部長訳する」ボタンは同じブロックにまとめ、ユーザーが最初に触る場所を迷わせない。
- 翻訳結果は入力欄の直下または右側に置き、入力から結果への流れを近く見せる。
- 3層出力は「1. 部長向け1行」「2. 稟議に書ける効果」「3. 想定ツッコミ」の順でカード化し、同じ変換結果のまとまりとして扱う。
- 辞書カードは「元の語」「部長訳」「例文」を1枚にまとめる。元語だけ、訳だけが離れて見えないようにする。
- 辞書追加カスタムは辞書一覧の近くに置き、「足りない言葉をその場で足す」行動につなげる。

### 整列
モバイルファーストでは縦並び、PCでは左右2カラムにする。

- 375px〜767px: ヒーロー、入力欄、結果、辞書一覧、辞書追加を1カラムで積む。
- 768px以上: 入力欄を左、翻訳結果を右に置く2カラム。左右の上端を揃え、貼る→見るの作業を同じ視線内に収める。
- 辞書一覧はCSS Gridでカードを並べる。モバイル1列、タブレット2列、デスクトップ3列。
- 3層出力のカードは見出し、本文、コピーボタンの位置を揃える。
- セクション幅は最大1120px程度に抑え、JTCの業務画面らしく読みやすい密度にする。

### 反復
同じ意味を持つUIは同じ形で繰り返し、軽い笑いがありつつ仕事中に開ける落ち着きにする。

- 辞書カードはすべて同じ構造にする: カテゴリラベル、元の語、矢印、部長訳、例文。
- 3層出力ブロックは番号、短い見出し、本文、コピーボタンを反復する。
- カラーパレットは紺をベースに、承認印の赤や蛍光付箋の黄色をアクセントとして反復する。
- ボタン、タグ、カード角丸、余白を統一し、雑多な辞書サイトではなく「社内で使う道具」に見せる。
- 文言のトーンは「笑えるが、ふざけすぎない」。例: `部長訳する`、`稟議に使う`、`ツッコミに備える`。

### 対比
「カタカナの未来」と「部長に通る言葉」の変換が主役なので、言葉同士の差を視覚的に強く出す。

- 元の語はやや冷たい青灰色、部長訳は濃紺または墨色で太く表示する。
- `DX → デジタル化`の矢印はアクセント色にし、変換された感覚を出す。
- CTAボタンは紺背景に白文字、ホバー時に承認印の赤を少し使う。
- 重要な3層出力は背景色を変え、辞書一覧よりも先に目に入るようにする。
- 注記や法務文は薄いグレー背景で控えめにし、主機能と情報の重みを分ける。

## 配色案
### 案A: JTCネイビー + 承認印レッド
仕事で開いても違和感がない紺を主役にし、判子・稟議・承認を連想する赤を小さく効かせる。

| 用途 | HEX |
| --- | --- |
| 背景 | `#F6F7F9` |
| メイン文字 | `#1D2433` |
| JTCネイビー | `#102A43` |
| 青灰の補助文字 | `#62748A` |
| 承認印レッド | `#C0392B` |
| 付箋イエロー | `#F2C94C` |
| カード背景 | `#FFFFFF` |
| 境界線 | `#D8DEE8` |

### 案B: 稟議グリーン + 紺 + 付箋イエロー
堅い社内資料感を少し和らげ、改善活動や現場掲示板の雰囲気を出す。

| 用途 | HEX |
| --- | --- |
| 背景 | `#F4F6F1` |
| メイン文字 | `#20231F` |
| 深緑 | `#1F4D3A` |
| JTCネイビー | `#17324D` |
| 付箋イエロー | `#FFD166` |
| 注意アクセント | `#B23A48` |
| カード背景 | `#FFFFFF` |
| 境界線 | `#D7DDCF` |

### 推奨
初期版は案Aを採用する。理由は、JTCの堅さ、部長向けの説得、承認印の笑いが最も伝わりやすいから。

## タイポグラフィ
### フォント方針
- 本文・UI: `system-ui`, `-apple-system`, `BlinkMacSystemFont`, `Segoe UI`, sans-serif。読みやすく、GitHub Pagesでも追加読み込み不要。
- 見出しの一部: `Hiragino Mincho ProN`, `Yu Mincho`, serifをアクセント的に使い、稟議書・社内文書らしさを出す。
- 数字・コード的な語: UIフォントのまま。辞書キーは太字にして視認性を上げる。

### サイズ階層
| 要素 | モバイル | デスクトップ |
| --- | --- | --- |
| H1 | 30px / 1.2 | 44px / 1.15 |
| セクション見出し | 22px / 1.3 | 28px / 1.25 |
| カード見出し | 17px / 1.35 | 18px / 1.35 |
| 本文 | 15px / 1.75 | 16px / 1.8 |
| 補助文 | 13px / 1.6 | 14px / 1.6 |
| ボタン | 15px / 1 | 16px / 1 |

### 表現ルール
- ヒーローの主語は「部長訳」または「カタカナの未来を、部長に通る言葉へ。」にする。
- 機能説明を長く置かず、入力欄と結果で価値を体験させる。
- 結果文は冗談に寄せすぎず、実際にコピペできる文体にする。

│ └────────────────────┘ │

┌────────────────────────┐
│ 翻訳結果                │
