# 実装計画書 — 部長訳

## フェーズ分け
### フェーズ1: 土台と辞書データ
完了条件:
- `index.html`、`style.css`、`script.js`、`data/dictionary.json`で動く静的構成がある。
- GitHub Pagesでそのまま公開できるファイル配置になっている。
- 初期辞書50語のうち、最低限の核となる語が入っている。
- 辞書データに`translation`、`summaryPhrase`、`effect`、`category`、`example`、`objections`が揃っている。

重点:
- darariがJTCで実際に通してきた言い換えを優先する。
- 技術的な正確性だけでなく、部長が判断しやすい業務語かを基準にする。

### フェーズ2: 変換ロジックと3層出力
完了条件:
- 入力文から辞書語を検出できる。
- 長い語を優先して置換できる。
- `PoC`、`poc`、`POC`などの表記揺れを吸収できる。
- 部長向け1行サマリー、稟議に書ける効果文、想定ツッコミ3つが生成される。
- 未登録語があっても処理が止まらない。

重点:
- 単なる置換結果で終わらせず、「反対される前提で先回りした文章」を出す。
- 3層出力はそのままTeams、Slack、メール、稟議下書きへ貼れる文体にする。

### フェーズ3: UI実装
完了条件:
- モバイルで入力、変換、コピーまで迷わず操作できる。
- デスクトップで入力と結果が左右2カラムになっている。
- 辞書一覧をカテゴリで見られる。
- 辞書追加カスタムで、その場の変換に語を追加できる。
- 入力値を`innerHTML`で描画せず、安全に表示している。

重点:
- 「笑えるけど業務中に開ける」見た目にする。
- 説明文を増やしすぎず、体験そのもので価値を伝える。

### フェーズ4: 品質調整と応募準備
完了条件:
- Chrome、Safari、Edgeで主要動作を手動確認済み。
- 主要サンプル文で3層出力が不自然ではない。
- OGP、title、description、注記が入っている。
- GitHub Pagesで公開URLを確認できる。
- READMEに公開URL、概要、使い方、応募作である旨が書かれている。

重点:
- コンテスト締切2026-06-30に間に合わせるため、React化やサーバー化はしない。
- v1で尖りが伝わることを優先し、URL入力やlocalStorage保存は後続に回す。

### フェーズ5: 公開後の辞書改善
完了条件:
- ユーザー反応から追加語候補を集める運用導線がある。
- 辞書100語化に向けたカテゴリ整理ができている。
- 追加した語が3層出力に効いているか確認する手順がある。

重点:
- 「面白い言い換え」だけを増やさない。
- 実際に上司説明、稟議、会議前の根回しに使える言葉だけを採用する。

## Codex実装指示テンプレート
```text
このリポジトリで「部長訳」のv1を実装してください。

前提:
- お悩み解決サイト選手権の応募作です。締切は2026-06-30です。
- 1ページの静的Webサイトにしてください。
- React、Next.js、バックエンド、外部APIは使わず、HTML/CSS/Vanilla JSで実装してください。
- GitHub Pagesで公開できる構成を維持してください。

プロダクトの核:
- 表面的な悩みは「カタカナ用語が上司に伝わらない」です。
- 本質は「自分は価値を分かっているのに、上司に伝わらないせいで前に進まない孤独」です。
- このサイトは翻訳ツールではなく、「未来を社内に持ち込むための変換装置」として作ってください。

実装する機能:
1. テキスト入力欄
   - 技術記事や企画メモを貼れるtextarea。
   - サンプル文を入れるボタン。

2. 辞書ベース変換
   - `data/dictionary.json`を読み込む。
   - カタカナIT/AI語をJTC語へ置換する。
   - 長い語を優先する。
   - 表記揺れを可能な範囲で吸収する。

3. 3層出力
   - 部長向け1行サマリー。
   - 稟議に書ける効果文。
   - 想定ツッコミ3つと先回り回答。

4. 辞書一覧
   - 元の語、部長訳、カテゴリ、例文をカード表示する。
   - カテゴリで絞り込めるようにする。

5. 辞書追加カスタム
   - 元の語、部長訳、例文、カテゴリを追加できる。
   - v1ではブラウザの現在セッション中だけ反映でよい。
   - 外部送信しない。

6. コピー機能
   - 3層出力それぞれにコピー用ボタンを置く。

デザイン:
- モバイルファースト。
- PCでは入力と結果を左右2カラム。
- 配色はJTCネイビーをベースに、承認印レッドをアクセントにする。
- 笑えるが、仕事中に開いても恥ずかしくない見た目にする。
- カードの角丸は控えめにする。

セキュリティ:
- 入力テキストを外部送信しない。
- ユーザー入力を`innerHTML`へ直接入れない。
- 解析タグ、広告タグ、外部APIは入れない。

完了条件:
- `index.html`をブラウザで開いて主要操作が動く。
- GitHub Pagesで公開可能。
- 仮置きコメントや空の実装が残っていない。
- `DX`、`アジャイル`、`PoC`、`AIエージェント`、`自動化`、`生産性向上`の変換が確認できる。
- 3層出力が空にならない。
```
